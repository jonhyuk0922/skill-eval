# 검수 기준 — 잘된 스킬의 다섯 축

스킬이 무너지는 자리는 거의 정해져 있다. 그 자리를 다섯 축으로 나눠두면, 내 스킬이 어느 축에서 짧은지 한눈에 보인다.

| 축 | 무엇을 보나 | 여기까지면 합격 |
|---|---|---|
| **① 명세 또렷함** | 입력 → 처리 → 출력 → 합격기준이 글로 또렷한가 | 네 칸이 각각 한 문장. "잘", "적당히", "알아서" 같은 모호한 말이 없다 |
| **② 검증 루프** | 진짜 사례로 돌려보고 고친 흔적이 있나 | 입력→출력 한 세트 + "이래서 이렇게 고쳤다" 한 줄 |
| **③ 진짜 업무성** | 내가 매주 실제로 반복하는 일인가 | "매주 월요일, 자막 → 블로그"처럼 구체적 |
| **④ 동료 이식성** | 설명만 읽고 옆자리가 바로 쓰나 | 설명이 "○○할 때 쓰세요" 꼴. 내 폴더 경로·내 사정이 안 박혀 있다 |
| **⑤ 임팩트** | 이 스킬로 반복이 실제로 줄었나 | "회당 30분 → 5분"처럼 수치로 말할 수 있다 |

> 다섯 줄 중 **가장 짧은 한 축**이, 오늘 고칠 자리다.

## 채점 척도

- **합격 (2점)** — 합격선 문장을 그대로 만족한다.
- **부분 (1점)** — 방향은 맞지만 합격선에 한 끗 모자란다. (예: 명세는 있는데 합격기준 칸이 비었다)
- **미달 (0점)** — 합격선을 못 만족하거나 해당 내용이 아예 없다.

합계 10점 만점. **합계보다 가장 낮은 축이 더 중요하다** — 그게 다음에 고칠 한 가지다.

## 합격을 가르는 건 보통 ① ② ③

④ 이식성·⑤ 임팩트는 ①②③이 서면 대개 따라온다. 그래서 처방은 ①②③에 먼저 집중한다.

## 3대 결함과 한 줄 처방

| 결함 | 증상 | 처방 (복붙용 한 줄) |
|---|---|---|
| **뭉뚱그린 명세** | "잘 정리해줘"처럼 입력·처리·출력·합격기준이 한 덩어리 | `이 스킬의 명세를 입력/처리/출력/합격기준 네 칸으로 쪼개고, 각 칸을 모호어 없이 한 문장으로 다시 써줘.` |
| **한 방에 끝** (검증 루프 없음) | 만들고 한 번도 안 돌려봄. 고친 흔적 0 | `내 실제 사례 하나로 이 스킬을 돌려보고, 어긋난 곳 한 군데를 고친 뒤 "이래서 이렇게 고쳤다"를 SKILL.md에 한 줄 남겨줘.` |
| **toy 예제** | 연습용 가짜 업무라 실제로 안 씀 | `이 스킬을 내가 매주 진짜 반복하는 업무(요일·트리거·산출물 명시)에 맞게 입력·예시·기준을 전부 바꿔줘.` |

## 신호를 두 갈래로 나눈다 — 자동(①④⑤) vs 사람표시(②③)

무너지는 자리라고 다 같은 게 아니다. 어떤 축은 글자로 잡히고, 어떤 축은 안 잡힌다.
그래서 `scripts/collect.py` 는 잡히는 것만 자동으로 신호를 주고, 못 잡는 건 **판정하지 않고 사람 확인란만** 내놓는다.

### 자동 신호 (①④⑤ — 기계가 비교적 잘 잡음)

- **①** 모호어 — `적당히`·`알아서`·`적절히`·`대충`, 그리고 독립 토큰 `잘`(잘못·잘된 같은 단어 속 '잘'은 제외) → 명세 또렷함 감점 신호
- **④** 하드코딩된 개인 경로 — `/Users/`·`/home/`, `C:\` 드라이브 경로, `바탕화면` → 동료 이식성 감점 신호
- **⑤** 줄어든 시간·횟수 수치(`분`·`회당`·`%`·`배`·`절반`) 없음 → 임팩트 감점 신호

### 사람이 직접 표시 (②③ — 기계가 못 가림)

②(검증 루프)·③(진짜 업무성)은 **단어가 우연히 있거나 없을 수 있어** 자동 판정하지 않는다.
collect.py 는 참고 숫자(예시 마커 몇 개, 반복어 몇 개)만 보여주고, 판정란은 빈 `[ ]` 로 남긴다.

- **②** 본문에 *실제 입력→출력 한 세트*와 *"이래서 이렇게 고쳤다" 흔적*이 있나? → 둘 다 있으면 합격, 하나면 부분, 둘 다 없으면 미달을 **사람이 표시**
- **③** 이게 정말 *요일·트리거·산출물로 말할 수 있는 반복 업무*인가? → **사람이 표시**

참고 숫자는 **근거가 아니다.** 최종 합격/미달은 합격선 문장을 잣대로 본문을 읽고 사람이 정한다.
